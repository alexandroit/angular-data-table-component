API reference

These inputs, column fields, and outputs are the levers that show up repeatedly across the Angular example tree.

Core inputs

Input Description
columnsDeclarative column definitions with selectors, sort rules, and optional templates.
dataThe row array to render for the current view.
paginationTurns on the built-in footer pagination UI.
paginationServerStops client slicing so the host component controls which rows are loaded.
sortServerLeaves sorting to the host component while still exposing the sort event payload.
themeSwitch between the default and dark presets.

Selection and expansion

Input Description
selectableRowsEnables checkbox or radio row selection.
selectableRowsSingleRestricts selection to a single active row.
selectableRowSelectedProgrammatically preselect rows from host rules.
selectableRowDisabledDisables rows without removing them from the dataset.
expandableRowsEnables expandable detail rows.
expandableRowTemplateAngular template used to render expanded row content.

Column capabilities

Field Description
selectorRead values through a property key or a row function.
formatReturn custom display text without a full template.
cellTemplateUse an Angular template for richer cell layouts.
sortFunctionOverride comparison logic for a single column.
omitHide a column while keeping it in the declarative config.
wrap / right / centerControl cell alignment and text wrapping declaratively.

Outputs

Output Description
sortChangeFires with the current column, direction, and visible rows.
selectedRowsChangeReturns the selection state and selected rows array.
pageChangeFires when the current page changes.
rowsPerPageChangeFires when the footer page size changes.
rowClickedFires when a row is clicked.
rowExpandToggledFires when a detail row opens or closes.
